Login

Configurazione applicativo IrisWeb

La configurazione di IrisWeb viene salvata nel file IrisWeb.ini, all'interno della cartella root dell'applicativo.

Per i dettagli sui parametri vedere la pagina B000 Configurazione applicativi web

Accesso all'applicativo

Il database proposto è quello scritto nel parametro di impostazioni operative DATABASE; dopo l'accesso all'applicativo viene scritta la stringa di connessione utilizzata, altrimenti rimane quella precedente.

E' possibile attivare il riconoscimento automatico dell'azienda in funzione del nome utente: vedere la pagina A008 Gestione aziende

Nota: Se IrisWeb è configurato nell'ambiente IIS, e la password dell'utente Oracle MONDOEDP risulta modificata, si rende necessario posizionare il file IrisWIN.ini contenente la password corretta nella cartella C:\WINDOWS\system32\inetsrv.

Al momento del login l'applicativo chiede di selezionare un profilo per l'utente (i profili assegnati ad un utente possono essere anche più di uno) se non viene selezionato alcun profilo, IrisWeb proporrà l'elenco dei profili disponibili. Se è disponibile un solo profilo per l'utente con cui si desidera effettuare l'accesso IrisWin utilizzerà automaticamente quel singolo profilo, anche se non viene indicato in modo esplicito.

La variabile PORT serve solo nel caso di utilizzo dell'eseguibile W000PIrisWEB.exe, e non nel caso più usuale che prevede l'installazione all'interno di WebServer quali IIS o Apache. Indica la porta sulla quale l'applicativo si pone in ascolto per le richieste delle sessioni client. Il valore di default è 5000.

La fase di login può essere automatizzata in due modalità differenti, denominate GET e POST.

Queste modalità sono particolarmente utili quando l'utente si è già precedentemente loggato, ad esempio alla propria intranet, e si desidera passare nome utente e password al server IrisWEB affinché ci si possa autenticare ad esso automaticamente, evitando di reintrodurre i dati di login.

Modalità GET

Con la modalità GET, di più semplice realizzazione, è possibile specificare nella querystring della url i parametri dell'account di connessione.

I parametri validi da utilizzare nella querystring sono questi (validi ovviamente anche per la modalità POST, descritta di seguito):

  • azienda
  • usr
  • pwd
  • profilo
  • database
  • loginesterno
  • wsauth

Ad es., per indicare automaticamente l'azienda AZIN si deve utilizzare il seguente metodo:

http://serverXX/irisweb/W000PIrisWEB_IIS.dll/?azienda=AZIN

Per specificare azienda, utente e profilo occorre invece impostare i parametri in questo modo:

http://serverXX/irisweb/W000PIrisWEB_IIS.dll/?azienda=AZIN&usr=NOMEUTENTE&profilo=RESPONSABILE

Un url così strutturato può ovviamente essere utilizzato sia nel codice HTML, per automatizzare il login, che direttamente nella barra degli indirizzi del browser per, ad esempio, essere memorizzato tra i siti preferiti.

Nota: i parametri passati nella querystring sovrascrivono le impostazioni di default specificate eventualmente nelle chiavi di registro.

Modalità POST

Con la modalità POST, invece, i parametri dell'account (quali nome utente e password) possono essere memorizzati in un form che invii tali dati al server di IrisWin in modo nascosto, ovvero con dei campi non visibili all'utente.

L'utilizzo di campi nascosti permette di rendere più gradevole la pagina da passare al server IrisWin. Un esempio di pagina di accesso in modalità POST può essere il seguente (il nome del form è ininfluente):

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">

<html>

<head>

<title>Login automatico di IrisWeb</title>

</head>

<body>

Autenticazione di IrisWeb in corso...

<form name="formIris" method="post" action="http://serverXX/IRISWEB/W000PIrisWEB_IIS.dll/">

<input type="hidden" name="azienda" value="AZIN">

<input type="hidden" name="usr" value="nomeUtente">

<input type="hidden" name="pwd" value="myPass">

<input type="hidden" name="database" value="ORCL">

<script type="text/javascript">

<!--

document.forms["formIris"].submit();

//-->

</script>

<noscript>

<input type="submit" name="SubIrisWeb" value="Avvia IrisWeb">

</noscript>

</form>

</body>

</html>

Nota: i parametri passati con il metodo POST sovrascrivono le impostazioni di default specificate eventualmente nelle chiavi di registro.

Recupero della password

E' possibile attivare la funzionalità di recupero password via mail mediante l'attivazione del parametro avanzato RECUPERO_PASSWORD.

Per il corretto funzionamento di questa funzione è necessario disporre di un server smtp aziendale, e aver impostato gli indirizzi email dei dipendenti sulle utenze web.

Nel form di login viene visualizzato un link denominato Recupera password che permette di effettuare l'invio di una mail all'indirizzo (o elenco di indirizzi) specificato nei dati dell'utenza web del dipendente.

Importante: questa funzione non è disponibile nel caso di single sign-on o autenticazione su dominio.

Personalizzazione delle pagine

Tramite l'applicativo di configurazione B000PConfigIrisWEB.exe è possibile indicare per ogni pagina quali componenti si vuole nascondere indicandoli nella sezione Campi da non visualizzare. L'esempio classico è quello di non visualizzare alcuni componenti della pagina di login quali Azienda e Database: in questo caso è necessario indicare W001FLogin come Pagina e come Componenti lblAzienda, edtAzienda, lblDatabase, edtDatabase, premendo per ciacuno il bottone con la freccia.

Ciascuna pagina inoltre permette di inserire un contenuto esterno all'inizio e alla fine del corpo della pagina. Il contenuto che si vuole inserire deve essere salvato nella cartella Template di IrisWEB, e deve essere chiamato P01_nomepagina.htm (se all'inizio) o P02_nomepagina.htm (se alla fine). Il file verrà quindi incluso nel file nomepagina.htm in corrispondenza dei tag {%P01_nomepagina.htm%}

Per esempio, se si vuole aggiungere del contenuto nella pagina W009 StampaCartellino, questa fa riferimento al template T009StampaCartellino.htm, per cui è possibile includere i files P01_T009StampaCartellino.htm o P02_T009StampaCartellino.htm. Il file P01_T009StampaCartellino.htm verrà incuso all'inizio del corpo della pagina, sotto l'intestazione, mentre il file P02_T009StampaCartellino.htm verrà incuso alla fine dela pagina.

Sicurezza

E' inoltre possibile rendere sicura la trasmissione dei dati al server di IrisWeb mediante una connessione HTTPS che permette di cifrare automaticamente i dati che viaggiano tra il browser dell'utente ed il server.

In questo modo nessuno potrà intromettersi nella comunicazione tra browser e server per cercare di carpire azienda, nome utente e password.

Per far questo, dopo essersi assicurati che il server IrisWeb sia opportunamente configurato, sarà sufficiente accedere a IrisWeb specificando nell'url HTTPS anziché HTTP, sia nella modalità GET sia in POST.

E' da notare che, nel caso in cui sia necessario utilizzare la modalità HTTPS, nel server di IrisWeb si dovrà necessariamente installare un Certificato Digitale SSL.

Gestione dei messaggi di log

L'applicativo può generare un log di determinati eventi, selezionabili dal programma di configurazione di IrisWEB: B000PConfigIrisWEB.exe

I messaggi riportati possono essere messaggi di errore, oppure relativi al comportamento dell'applicativo relativamente al consumo di memoria e alle connessioni al database utilizzate, ecc.

Nel momento in cui si attiva uno o più elementi di log, tutti i messaggi relativi vengono registrati sul database se possibile, altrimenti su un file denominato W000.log e posizionato nella stessa directory dell'applicativo web.

In situazioni di funzionamento normale si consiglia di non attivare nessuno di questi log, mentre se si vuole tracciare l'applicativo per trovare eventuali errori o anomalie, è bene includere tutte le stringhe, e poi valutare di conseguenza se togliere quelle superflue.

Ciascuna riga di log riporta fra parentesi angolari le prime due lettere del tipo di messaggio registrato (per es. <Er> per Errore, <Ac> per Accesso, ecc...).

Gestione del timeout della sessione

Per gestire la terminazione della sessione, quando questa è rimasta attiva perché l'utente ha chiuso il browser senza eseguire il logout, è previsto un timeout in minuti, oltre il quale la sessione viene chiusa automaticamente. In questo caso, se l'utente ha ancora il browser attivo e utilizza le funzionalità dell'applicativo, verrà visualizzata una pagina di errore, e l'utente dovrà necessariamente richiamare la home page dell'applicativo.

Il timeout di default è impostato a 30 minuti per l'operatore IrisWin, e a 5 minuti per il dipendente.

Per ridefinire tali valori, intervenire sulle impostazioni TIMEOUT_OPERATORE e TIMEOUT_DIPENDENTE indicando i minuti desiderati.

Il timeout corrispondente viene assegnato alla sessione dopo essere stati autenticati, e quindi dopo che l'account è stato riconosciuto come operatore o dipendente.

Il timeout predefinito al momento del login, prima dell'autenticazione, viene impostato come il minimo fra i due valori di cui sopra.

Gestione delle connessioni al database

Ogni sessione web si collega ad un pool di sessioni Oracle gestite dall'applicativo stesso. Il numero di sessioni Oracle disponibili dipende dalle sessioni web attive, dedicando ad ogni sessione Oracle tutte le sessioni web possibili nei limiti del numero massimo di cursori gestibili per ogni sessione Oracle.

Per default, viene assunto come utilizzo medio 10 cursori per sessione web, e considerando 1000 come numero massimo numero di cursori per sessione Oracle, ciascuna può gestire fino a 100 sessioni web.

Poichè l'utilizzo effettivo dei cursori dipende dal numero di utenti attivi contemporaneamente e dalle funzionalità usate, è possibile ridefinire questi parametri, intervenendo sulle impostazioni di sistema CURSORI_SESSIONE e MAX_OPEN_CURSORS indicando rispettivamente i cursori (ipotizzati) utilizzati per ogni sessione e il parametro OPEN_CURSORS dell'istanza Oracle.

Poichè al momento del login la procedura di autenticazione utilizza un altro pool di connessioni Oracle, diverso da quello descritto prima, è possibile allo stesso modo intervenire sui cursori usati in questa fase per ciascuna connessione web, tramite la chiave CURSORI_LOGIN. Il valore di default è 12, e rappresenta gli effettivi cursori utilizzati, ma il valore è comunque modificabile nel caso si verificassero problemi di connettività al database.

Controlli avanzati dell'applicativo

Tramite l'impostazione PARAMETRI_AVANZATI, è possibile parametrizzare alcuni comportamenti dell'applicativo, nel caso in cui le normali impostazioni dovessero dare problemi di funzionamento.

Si veda la pagina B000 Configurazione applicativi web

Questi parametri sono attivabili solo dietro confronto col fornitore.

Termine della sessione di lavoro

Cliccando sul link Esci, il comportamento di default prevede di richiamare una pagina bianca indicante la chiusura della sessione. In alternativa, è possibile valorizzare l'impostazione HOME, indicando l'URL della pagina a cui si vuole accedere dopo aver cliccato su Esci.
